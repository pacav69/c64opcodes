BVS

Branch on oVerflow Set

Operation: Jump if V = 1



Mode Syntax HEX LEN TIME
Relative BVS $nn $70 2 2+
N V — B D I Z C
               

+ A branch not taken requires 2 machine cycles.
+ A branch requires 3 cycles if taken, plus 1 cycle if the branch cross a page boundary.

If Overflow flag = 1, jump the number of bytes expressed by the opcode's next byte. This byte will be interpreted as a signed byte (in Two's complement), allowing to jump forward (127 bytes) or backwards (128 bytes). The assembler will automatically calculate this displacement when we indicate a label as the destination for the jump, but we would need to understand Two's complement numbers in order to assemble relative branchs manually.

Internally jumping consists of modifying the Program Counter register (PC), wich is the 16 bit register that contains the address for the next instruction to be executed. Basically this is what the instruction will do: it will add or substract the proper number of bytes expressed by the operand to the PC if the condition V flag = 1 is true.

For jumps greater than the range of one signed byte we must use the JMP instruction. Considering that all the conditional branch instructions are using Relative Addressing and JMP uses Absolute Addressing, if we need to jump to a far place into our program conditionally we can play with conditional branching points pointing to absolute jumps.

Note: There is no BRA (Branch Always) instruction but it can be easily emulated by branching on the basis of a known condition. One of the best flags to use for this purpose is the Overflow flag which is unchanged by all but the addition and subtraction operations.
