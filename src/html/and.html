The negative flag is set if the result is negative, i.e. has its most significant bit set.
The zero flag is set if the result is zero, or cleared if it is non-zero.

Performs a logical AND between the bits of the accumulator and that of the operand specified (next byte of the opcode in immediate addressing). The result of AND operations will be 1 only when both bits compared are 1, else will be always 0.

A common usage for this instruction is to deactivate (reset) a particular bit or set of bits within a bit mask, i.e. puting zeros in the bit places we want to reset (bit = 0) and ones in the places we want to be unaltered. To activate bits we will use ora with a the contrary bit patter for the mask: one in the bits we need to activate and zeros in the bits we need to be unaltered.

Example, if we want to deactivate bit 5 (and only this bit) in the variable "value":

    lda value        // Loads value in Accumulator
    and #%11011111   // Mask: zero in bit 5, ones in the others.
    sta value        // Updates value (with bit 5 off)
