Strangely enough, an instruction that does nothing could be more useful than it seems at first glance. This instruction, for the simple fact that spends execution clock cycles, namely 2, can be used to synchronize subroutines to reach a given number of cycles. But the main task of this particular instruction is to facilitate debugging. It is very convenient to insert a few NOP instructions to "turn off" temporarily sections of code that we think may be causing problems or for patching sections of code.
