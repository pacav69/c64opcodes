ORA

Operation: A | M => A

+ add 1 cycle if page boundary crossed.
The negative flag is set if the result is negative, i.e. has it's most significant bit set.
The zero flag is set if the result is zero, or cleared if it is non-zero.

Performs a "logic OR" between accumulator and operand in memory. The result of the operation will be 1 if any of the checked bits contains a 1, so the result will only be zero is both bits are zero.

A very common usage for this instruction is to activate certain bits given a pattern (called mask). To deactivate (put bits to zero) we should use the instruction and with a proper mask, that is inserting zeros in the positions we need to deactivate.

Example: Activate bit 5 (and only this bit) in the variable "value".

  lda value        // Loads value in Accumulator
  ora #%00100000   // mask: one in bit 5, zeros in the other bits.
  sta value        // Updates value (with bit 5 on)

In this example, let's put that value = %11010010 (would have bit 5 = 0)

When executing ORA, and saving it in value, the result will be %11110010. There would only be changed the bit position where we had put a one (in the fifth position).
