EOR

Exclusive - OR memory with accumulator

Operation: A ^ M => A

+ add 1 cycle if page boundary crossed.
The negative flag is set if the result is negative, i.e. has its most significant bit set.
The zero flag is set if the result is zero, or cleared if it is non-zero.

Performs a "logic exclusive OR" operation between the bits of the accumulator and the operand in memory. The result of EOR will be 1 if both are 1 or 0 (0 EOR 0 = 1, 1 EOR 1 = 1), and will result in 0 in all the other cases.

A very common use for EOR is to create switches that toggle between two states. Also another common and interesting use of this instruction is to invert numbers, effectively obtaining this way the one's complement.

Example of a switch, using EOR:

        lda value       // Loads a value in accumulator (init. value=0)
        eor #$FF        // Invert bits of accumulator
        sta value       // Updates value ($00 or $FF alternatively)
        bne state1 
state0: ...             // Do something for state0
        rts
state1: ...             // Do something for state1
        rts
