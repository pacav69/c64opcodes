BCS

Branch on Carry Set

Operation: Jump if C = 1

Mode Syntax HEX LEN TIME
Relative BCS $nn $B0 2 2+
N V — B D I Z C
               

+ A branch not taken requires 2 machine cycles.
+ A branch requires 3 cycles if taken, plus 1 cycle if the branch cross a page boundary.

If Carry flag = 1, jump the number of bytes expressed by the opcode's next byte. This byte will be interpreted as a signed byte (in Two's complement), allowing to jump forward (127 bytes) or backwards (128 bytes). The assembler will automatically calculate this displacement when we indicate a label as the destination for the jump, but we would need to understand Two's complement numbers in order to assemble relative branchs manually.

Internally jumping consists of modifying the Program Counter register (PC), wich is the 16 bit register that contains the address for the next instruction to be executed. Basically this is what the instruction will do: it will add or substract the proper number of bytes expressed by the operand to the PC if the condition Carry flag = 1 is true.

This instrucion is useful for branching if a number is greater than another. To do this we'll check with a compare instruction a number against another stored somewhere in memory. If the result sets the carry (C=1) it will be that the verified number is greater than the other one. This is because internally compare instructions are really substracting (in fact, adding two's complement signed numbers) and then they update the flags acording to the result obtained (the result on itself doesn't matter really).

Example: jump if value1 > accumulator:

                  lda #$20            // Loads the number $20 in the
                                      //  accumulator.
                  cmp value1          // compare $20 with value1
                  bcs value1_greater  // if > then jump to
                                      //  value1_greater.
                  rts
  value1_greater: nop                 // we reach here if value1 > $20

For jumps greater than the range of one signed byte we must use the JMP instruction. Considering that all the conditional branch instructions are using Relative Addressing and JMP uses Absolute Addressing, if we need to jump to a far place into our program conditionally we can play with conditional branching points pointing to absolute jumps.

Also if we need to make our routines 100% relocatable (in memory), we need to use the relative branching instructions.

If we need "to force" a jmp (Absolute), to be a relative branch, we can change the jmp by any other relative branch instruction, previously setting up the flag that controls that instruction. Also, because the carry flag is being used very often than for example the overflow flag, is preferable to force relative jumps with bvc, but you could use any relative branch you want, only taking in mind that the flag that controls the instruction can be easily altered at your convenience.

Example using bcc for relocatable routines:

         sec          // C = 1
         bcs jump     // This is a relative jmp (forced)
                      // Which makes this part of the routine
                      //  relocatable.
  jump:  jmp jump2    // this jmp is absolute
                      // so this part couldn't be relocated without
                      //  modifications.
  jump2: rts

Note: There is no BRA (Branch Always) instruction but it can be easily emulated by branching on the basis of a known condition. One of the best flags to use for this purpose is the Overflow flag which is unchanged by all but the addition and subtraction operations.
